# 🍳 Кулинарная книга (Cookbook API)

REST API для управления рецептами. Реализован на **FastAPI** с асинхронной **SQLAlchemy** и **SQLite**.

## 📚 Описание

Сервис позволяет:
- Получать список всех рецептов с сортировкой по популярности (просмотры) и времени приготовления.
- Просматривать детальную информацию о конкретном рецепте (счётчик просмотров увеличивается).
- Добавлять новые рецепты с ингредиентами.

## 🛠️ Стек

- **FastAPI** – веб-фреймворк
- **SQLAlchemy (async)** – ORM для работы с БД
- **SQLite / aiosqlite** – база данных
- **Pydantic** – валидация и сериализация
- **Uvicorn** – ASGI-сервер
- **Pytest** + **httpx** (или встроенный `TestClient`) – тестирование

## 📁 Структура проекта

├── main.py # Инициализация FastAPI, эндпоинты
├── database.py # Асинхронный движок, получение сессии, Base
├── models.py # Модели SQLAlchemy (Recipe, Ingredient, связь M2M)
├── schemas.py # Pydantic-схемы для запросов и ответов
├── crud.py # Асинхронные функции взаимодействия с БД
├── requirements.txt # Зависимости
├── tests/
│ └── test_main.py # Тесты API
├── cookbook.db # Файл БД (создаётся автоматически при первом запуске)
└── README.md

Запусти сервер:
python -m uvicorn main:app --reload
Сервер стартует на http://127.0.0.1:8000.

Документация API:
Swagger UI: http://127.0.0.1:8000/docs

ReDoc: http://127.0.0.1:8000/redoc

🧪 Запуск тестов
Тесты используют асинхронную SQLite в памяти, поэтому не портят основную базу данных.

Проверяют:
- Создание рецепта через POST /recipes
- Получение списка через GET /recipes
- Увеличение просмотров при повторных GET /recipes/{id}
- 404-ошибку при несуществующем рецепте

🔧 Особенности реализации:
- Асинхронность на всех уровнях (FastAPI + AsyncSession SQLAlchemy).
- Автоматическая документация благодаря Pydantic-схемам и аннотациям FastAPI.
- Валидация данных: cooking_time должен быть > 0, ингредиенты – непустой список строк, даты в прошлом примере (если бы использовались) проверяются через Pydantic.
- Сортировка: views DESC, cooking_time ASC.
- Управление просмотрами: счётчик увеличивается атомарно при каждом запросе деталей.
